# CircuitProxy — POST_MVP_PLAN

Возможности, осознанно вырезанные из MVP. Не начинать, пока Этапы 1–5 не завершены
и стабильны. Порядок — примерный приоритет.

## 1. Weighted balancing

Веса на backend'ах (например, `weight: 3` получает втрое больше трафика). Замена
чистого round-robin на weighted round-robin или smooth weighted round-robin
(алгоритм nginx). Конфиг расширяется полем `weight` на backend.

## 2. Sticky sessions

Привязка клиента к конкретному backend'у (по cookie или по хэшу client IP —
consistent hashing). Нужен для stateful-backend'ов. Взаимодействие с health-check:
что делать при падении «залипшего» backend'а (fallback + перепривязка).

## 3. TLS termination

Приём HTTPS на стороне прокси (`tls.Config`, сертификаты из конфига), проксирование
на backend'ы по plain HTTP или re-encrypt. Опционально — ACME/автосертификаты
(`golang.org/x/crypto/acme/autocert`) — это первая внешняя зависимость проекта.

## 4. Множественные backend-группы + маршрутизация

Несколько пулов backend'ов с маршрутизацией входящих запросов по пути/хосту
(`/api/*` → пул A, `/static/*` → пул B; или по `Host`-заголовку). Каждый пул —
свой набор circuit breaker'ов и health-check'ов. Конфиг становится списком
route-правил вместо плоского списка backend'ов.

## 5. Прочие улучшения устойчивости

- **Half-open с ожиданием** вместо fast-fail: не-пробные запросы во время half-open
  ждут результат единственного пробника (короткий bounded wait) вместо немедленной
  ошибки. Улучшает user-facing latency ценой сложности (нужна аккуратная отмена по
  таймауту, чтобы не копить горутины).
- **Скользящее окно ошибок** вместо счётчика подряд идущих (rolling window / EWMA
  error rate) для порога breaker'а — устойчивее к спорадическим ошибкам.
- **Outlier detection** — автоматическое временное исключение backend'а по повышенной
  латентности, а не только по факту ошибок.
- **Динамическая перезагрузка конфига** (SIGHUP) без рестарта.
- **Полноценный Prometheus-экспортер** (`prometheus/client_golang`) вместо текстового
  `/metrics` — вторая внешняя зависимость.
- **Graceful shutdown** входящих соединений (drain) при остановке прокси.
